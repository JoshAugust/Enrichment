## 3. ZoomInfo API — Research Findings

### Access Model

ZoomInfo's API is **enterprise-gated** — there is no public self-serve developer tier or free trial. Access requires:

1. An active ZoomInfo subscription (Enterprise tier)
2. Admin assignment of the DevPortal subscription
3. Authentication via Client ID + Client Secret (OAuth2)

**Docs:** https://docs.zoominfo.com (legacy: https://api-docs.zoominfo.com)

### Pricing (2024–2025)

| Tier | Annual Cost | Includes |
|---|---|---|
| Professional | ~$15,000/year | 5,000 credits, basic access |
| Advanced | ~$25,000/year | 10,000+ credits |
| **API (Search/Enrich)** | **~$50,000+/year** | Prospecting + lead gen API |
| Enrichment only (HubSpot) | ~$5,000/year | Enrich existing records only |

**This is expensive.** There is no free tier or trial for the API. Credits are consumed per unique record exported/revealed.

### Key Endpoints

| Endpoint | Description |
|---|---|
| `POST /search/company` | Search for companies by attributes (industry, size, revenue, etc.) |
| `POST /search/contact` | Search for contacts (title, seniority, department, etc.) |
| `POST /enrich/company` | Enrich a known company (lookup by domain/name/ID) |
| `POST /enrich/contact` | Enrich a known contact (lookup by email/LinkedIn/name+company) |
| `POST /enrich/contact/bulk` | Bulk enrich up to 2,500 contacts per request |
| `POST /enrich/company/bulk` | Bulk enrich companies |

**Rate limits:** Up to 1,500 requests/minute on standard Enrich endpoints.

**Authentication:**
```
POST https://api.zoominfo.com/authenticate
Body: { "client_id": "...", "private_key": "..." }
→ Returns: { "jwt": "Bearer token" }
```

### Recommendation for Corgi

ZoomInfo is **not viable as a free or low-cost enrichment source** for Corgi's current stage. Minimum viable entry is ~$5K/year for enrichment-only. If the pipeline ever needs it:
- Set `ZOOMINFO_CLIENT_ID` and `ZOOMINFO_PRIVATE_KEY` env vars
- Authenticate via JWT before each session (tokens expire)
- Build a source module at `backend/src/research/sources/zoominfo.js`

Alternatives at lower cost: Hunter.io (built ✅), Apollo.io (has a free tier), Clearbit (now part of HubSpot), Lusha.

## 4. Hunter.io Integration — Built ✅

### What was built

**File:** `backend/src/research/sources/hunter.js`

A full enrichment source implementing the standard `enrich(entityType, entityId, existingData)` interface.

### Capabilities

**For companies (`entityType = 'company'`):**
- Calls Hunter.io Domain Search API (`/v2/domain-search`)
- Returns all discovered email addresses as `contacts[]`
- Surfaces the detected email pattern (e.g., `{first}.{last}`)
- Filters out low-confidence results (< 50) and generic/role emails (info@, contact@)
- Skips gracefully if no domain is available

**For contacts (`entityType = 'contact'`):**
- Calls Hunter.io Email Finder API (`/v2/email-finder`)
- Uses first + last name + company domain to find specific email
- Normalizes confidence score from Hunter's 0–100 scale to 0–1

### Pipeline integration

`enrichment-pipeline.js` was updated to:
1. Register `'hunter'` in the `sources` object
2. Call `applyHunterContacts(companyId, result.contacts)` after Hunter runs — this upserts newly discovered contacts into the `contacts` table (new contacts inserted; existing contacts updated with email if missing; duplicates safely skipped)

### Free Tier Limits

> ⚠️ Hunter.io free tier: **25 searches/month** total (domain-search + email-finder combined).  
> Paid plans start at ~$49/month (500 searches) and ~$149/month (5,000 searches).  
> Monitor usage at: https://hunter.io/dashboard

## 5. Environment Variables Required

| Variable | Source | Required? | Notes |
|---|---|---|---|
| `HUNTER_API_KEY` | Hunter.io API keys page | **Yes** (for Hunter source) | Get from https://hunter.io/api-keys |
| `ZOOMINFO_CLIENT_ID` | ZoomInfo DevPortal | Optional | Only if/when ZoomInfo subscription acquired |
| `ZOOMINFO_PRIVATE_KEY` | ZoomInfo DevPortal | Optional | RSA private key for JWT auth |

### Setting env vars

Add to your `.env` file at `backend/.env` (or wherever the app loads env from):

```env
# Hunter.io — email discovery via domain search
# Free tier: 25 searches/month. Get key at https://hunter.io/api-keys
HUNTER_API_KEY=your_hunter_api_key_here

# ZoomInfo — NOT YET INTEGRATED (expensive, ~$50K/year for full API)
# ZOOMINFO_CLIENT_ID=
# ZOOMINFO_PRIVATE_KEY=
```

If `HUNTER_API_KEY` is not set, the Hunter source **gracefully skips** (returns `skipped: true`, does not throw) — zero breaking behavior.
